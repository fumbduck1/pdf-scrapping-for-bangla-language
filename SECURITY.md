# Security Policy

## Supported Versions

Security fixes are applied to the latest main branch. Older snapshots or forks are not maintained.

## Reporting a Vulnerability

1. **Do not file public issues.**  
2. Use the **“Report a vulnerability”** link in the repository’s **Security** tab to create a private GitHub Security Advisory.  
3. If that is not available, email the maintainers at: `security@fumbduck1.com` with a clear description, reproduction steps, impact, and any suggested mitigations.

## Expectations

- You will receive an acknowledgment within **3 business days**.  
- We aim to provide a remediation plan or mitigation guidance within **10 business days** for confirmed issues.  
- Please allow time for fixes to be prepared and released before any public disclosure.

## Scope

Security issues related to code execution, data integrity, denial of service, and sensitive information exposure in this project are in scope. Documentation issues and feature requests are not treated as security vulnerabilities.
